# Summary Report Template

Use this template for `04-summary.md`.

This report should synthesize `01-product-understanding.md`,
`02-market-and-strategy.md`, and `03-build-mvp.md` into a concise
management-facing summary after a light consistency check.

## Summary Report

### 1. One-Paragraph Bottom Line

- What the product is in one sentence
- Whether it appears feasible to build a similar product
- What the main caveat or caution is

### 2. Core Judgment

Organize the main judgment in a pyramid structure:

- top-level conclusion first
- then `2-4` supporting reasons
- then the most important evidence or constraint behind each reason

### 3. Product Snapshot

Summarize only the most decision-relevant facts:

- target users
- core workflow
- business model clues
- confirmed dependency or ecosystem constraints

### 4. Competitive Edge And Core Risk

Compress the strongest moat-like factors and the corresponding risks into
paired bullets. Favor decision value over coverage.

### 5. Build Feasibility

Summarize:

- what looks straightforward
- what looks deceptively hard
- what depends on external partners, compliance, or operating maturity

### 6. Cost And Operating Burden

Summarize the highest-signal cost points:

- explicit third-party cost surfaces
- hidden costs such as reserves, returns, disputes, KYC remediation,
  support, or delayed payout
- items that cannot be defensibly quantified yet

### 7. Consistency Check Notes

Keep this short. Only note issues that materially affect trust in the
summary:

- conclusions that are well-supported across `01/02/03`
- claims excluded from the summary because of weak support or conflict
- any source or wording mismatch that should be treated cautiously

### 8. Source Pointers

Do not repeat every source. Point to the most important report files and
the most important official URLs.
